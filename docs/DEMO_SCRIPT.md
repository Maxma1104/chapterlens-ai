# Two-minute demo script

## 0:00–0:15 — Problem

Show the landing page.

“AI manuscript feedback is fast, but it is hard to audit. ChapterLens is built around one rule: editorial claims must show the exact words that support them.”

## 0:15–0:35 — Input

Open **Review desk**, choose **Load example**, and point to the title, text limit, privacy note, and `.txt` upload.

“The MVP handles one chapter or long excerpt. The bounded scope makes cost and failure behavior predictable.”

## 0:35–0:55 — Pipeline

Run the review and let the five progress stages appear.

“The system separates extraction, editorial reasoning, and citation validation. Invalid evidence cannot reach the report.”

## 0:55–1:25 — Result

Show the summary, continuity scan, pacing, then characters and timeline.

“The output is structured rather than a wall of generated prose, so each capability can be tested and each claim can be addressed.”

## 1:25–1:40 — Evidence

Click citation `1` and show the highlighted source.

“Offsets are rebuilt on the server from exact quotations. If the quote is missing, the claim is removed. If the summary loses support, ChapterLens refuses.”

## 1:40–1:55 — Production proof

Open history, then briefly show the GitHub evaluation report and CI workflow.

“Signed-in data is isolated with row-level security. The repository includes atomic quotas, cost logging, monitoring, tests, and 50 evaluation cases.”

## 1:55–2:00 — Close

“ChapterLens is not trying to replace an editor. It makes AI-assisted reading faster to verify and safer to challenge.”
